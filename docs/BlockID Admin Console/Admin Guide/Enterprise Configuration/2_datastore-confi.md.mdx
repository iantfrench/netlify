# Datastore Configuration

## Overview 

In Datastore Configuration, the specified fields are used to show user details within the Admin Consoleâ€™s User Profile or User Management screen. These scope fields are mapped to your datastore (AD/LDAP) directory fields. Meanwhile, if you have changed any of your directory field names at your end then you can update those names mapped to the respective scope fields under the Datastore Configurations screen.

To access the Datastore Configuration:
1. Login to **BlockID Admin Console**, navigate to ***Administration Console** >  **Enterprise Configuration** > **Datastore Configuration***.

![DatastoreConfigurationscreenSN.jpg](https://cdn.document360.io/aeae7c88-d2f4-4a9c-bba9-c4f368260d78/Images/Documentation/DatastoreConfigurationscreenSN.jpg)

3. In **Datastore Configuration**,  map the appropriate **Scopes** fields to your **Directory Attributes** fields:
    * `firstname` to `<FIELD_NAME FOR FIRSTNAME>`.
    * `locality` to `<LOCALITY>`.
    * `middlename` to `<MIDDLE_NAME>`.
    * `lastname` to `<LAST_NAME>`.
    * `phone` to `<PHONE_NUMBER>`.
    * `postalcode` to `<POSTAL_CODE>`.
    * `state` to `<STATE>`.
    * `email` to `<EMAIL_ADDRESS>`.

4. Click **Save Datastore**.
